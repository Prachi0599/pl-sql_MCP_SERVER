HOW TO USE
  - Start the chat client from the project root:  python chat.py
  - Type any question below. You get a plain-English answer.
  - WRITE questions stage an approval and ask "yes/no" first - reply "no" to keep
    the database unchanged, or "yes" to actually apply.
  - "no change / already exists" replies are CORRECT behaviour (no-op detection).
  - Type /raw to see the underlying JSON or generated SQL for any answer.
  - Type /pending to see staged approval requests; /quit to exit.

REAL SEED IDENTIFIERS (use these so questions return data)
  customer number      : CUST000115 .. CUST010114   (e.g. CUST000122, CUST000200)
  account number       : ACC000071 .. ACC010070     (e.g. ACC000123, ACC000071)
  invoice number       : INV00000123 (and similar)
  invoicing company    : INV0001, INV0002, INV0003 (countries: India, UK, USA, Germany)
  customer types       : CORP, ENT, GOV, SMB, WHOLESALE
  product code         : PROD0001 .. PROD0100        (e.g. PROD0048)
  product types        : MPLS, INTERNET, IPVPN, SDWAN
  currencies           : USD, GBP, INR, EUR
  account statuses     : ACTIVE, INACTIVE, SUSPENDED
  bill statuses        : GENERATED, PAID, PENDING, OVERDUE
  source systems       : USAGE_COLLECTOR, NETWORK_GATEWAY, BILLING_ENGINE
  service req priority  : HIGH, MEDIUM, LOW            (CRITICAL is auto-mapped to HIGH)
  service req type      : DATA_FIX, BILLING_ADJUSTMENT, RCA, QUERY, OTHER
  adjustment type       : CREDIT, DEBIT, DISPUTE, WAIVER
  customer note type    : GENERAL, BILLING, ESCALATION, RCA, TECHNICAL
  packages (9)         : ACCOUNT_PKG, BILLING_PKG, BILLING_ADJUSTMENT_PKG,
                         CUSTOMER_PKG, LOAD_MONITOR_PKG, MCP_SECURITY_PKG,
                         METADATA_PKG, SERVICE_REQUEST_PKG, USAGE_ANALYTICS_PKG
